FlashTrade Backend Postman Test Routes
=====================================

Base URL
- http://localhost:5000 (or your configured server port)

Common Headers
- Content-Type: application/json (for POST/PUT/PATCH with JSON body)
- Authorization: Bearer <access_token> (only for protected routes)

Token Notes
- Get access token from: POST /auth/login
- Refresh token route: POST /auth/refresh-token


AUTH ROUTES (/auth)
-------------------
1) POST /auth/signup
   - Auth header: NO
   - Role: Public

2) POST /auth/login
   - Auth header: NO
   - Role: Public

3) POST /auth/bootstrap-admin
   - Auth header: NO
   - Role: Public (initial setup)

4) POST /auth/refresh-token
   - Auth header: NO
   - Role: Public (uses refresh token flow)

5) POST /auth/logout
   - Auth header: NO
   - Role: Public (based on current route definition)

6) POST /auth/forgot-password
   - Auth header: NO
   - Role: Public

7) POST /auth/reset-password/:token
   - Auth header: NO
   - Role: Public

8) PUT /auth/change-password
   - Auth header: YES
   - Role: Any authenticated user

9) PUT /auth/update-profile
   - Auth header: YES
   - Role: Any authenticated user

10) GET /auth/profile
    - Auth header: YES
    - Role: Any authenticated user

11) GET /auth/users
    - Auth header: YES
    - Role: ADMIN only

12) POST /auth/admins
    - Auth header: YES
    - Role: ADMIN only

13) PATCH /auth/admins/:userId/promote
    - Auth header: YES
    - Role: ADMIN only

14) GET /auth/sellers/pending
    - Auth header: YES
    - Role: ADMIN only

15) PATCH /auth/sellers/:userId/approve
    - Auth header: YES
    - Role: ADMIN only

16) GET /auth/sellers/approved
    - Auth header: YES
    - Role: ADMIN only

17) GET /auth/sellers/:userId
    - Auth header: YES

18) PATCH /auth/sellers/:userId/status
    - Auth header: YES

Quick Postman Setup
-------------------
1) Create environment variable: baseUrl = http://localhost:5000
2) Login first and save token from /auth/login as: accessToken
3) For protected routes set header:
   Authorization: Bearer {{accessToken}}
4) For JSON requests set header:
   Content-Type: application/json
